## Completed Changes

### Runtime timing

- Replaced raw `TICK_SECONDS = 0.2` with `addon.UPDATE_INTERVALS.fifth_sec`.
- Left `FILL_TEST_TICK_SECONDS = 0.05` module-local because it drives smoother fill-test animation.

### Layout efficiency

- Moved the clean-layout guard to the top of `M.apply_layout()`.
- Stable refresh ticks now return before DB/frame lookup and before layout-signature construction.
- `render_fill_test()` did not need separate changes because its repeated layout call now takes the fast path and its other setters already guard stable state.

### DB/default flow

- Simplified `ADDON_LOADED` so `get_db()` owns first default application and normalization.
- The loader still ensures top-level `Ls_Tweeks_DB` exists before calling `get_db()`.

### Charge/runtime readability

- Renamed spell-charge fallback local from `current` to `spell_current`.
- Renamed `normalize_power_value()`'s scaling-reference parameter from `max_slots` to `slot_count`.
- Made the ignored second return from `get_gliding_state()` explicit in `M.refresh()`.

### Slider callbacks

- Updated `CreateSliderWithBox()` to pass the current numeric value to callbacks.
- Existing zero-argument callbacks remain compatible.
- Updated Skyriding Vigor slider callbacks to pass the received value directly into `M.set_db_value()`.

### Decor sizing

- Added `DECOR_ATLAS = "dragonriding_vigor_decor"`.
- Added cached decor atlas sizing through `C_Texture.GetAtlasInfo()`.
- Removed live decor texture `GetWidth()`/`GetHeight()` reads from layout.

### Move mode ticker

- Changed `needs_progress_updates` so move mode does not request recurring progress ticks.
- Important detail: the condition explicitly excludes move mode because fake move-mode charge data uses a nonzero duration.

### Defaults naming

- Renamed the local staging table in `sv_defaults.lua` from `M` to `defs` to avoid confusing it with `addon.skyriding_vigor`.
- Exported defaults shape is unchanged.

### Drag robustness

- Added `frame._is_dragging` guard to ignore duplicate drag-start events while dragging.
- Cleared `_is_dragging` in `OnDragStop`.

## Second-Pass Findings

N1 - not an issue: The `apply_layout()` early-return guard `not M._layout_dirty and M._layout_signature` is correctly written as a conjunction; both conditions must hold to skip work.

N5 - resolved: Removed dead visual-layer branch scaffolding for `DRAW_BACKGROUND_IN_FRONT_OF_FRAME` and `DRAW_FILL_IN_FRONT_OF_FRAME`; both constants were permanently false.
