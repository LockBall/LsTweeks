## Summary

- Aura Frames currently use ordinary rectangular `BackdropTemplate` frame backgrounds.

- WoW does not expose a built-in plain colored rounded-rectangle or chamfered-rectangle primitive for frames.

- `BackdropTemplate` supports background and border textures, but it does not have a shape flag for rounded or 45-degree chamfered corners.

- Blizzard-style rounded/chamfered panels are usually texture-driven, often through NineSlice-style assets, not geometry generated by the frame API.

- The chamfer setting was removed for now because the Lua texture-strip approach became visually fiddly and harder to maintain than expected.


## What Was Tried

- A per-frame `Chamfer BG` option was briefly prototyped for preset and custom Aura Frames.

- The first prototype kept the frame hitbox rectangular and attempted to draw a visibly chamfered background/border with addon-owned texture strips.

- The initial renderer produced blocky corner artifacts because it cut out the square corner area and relied on rotated border textures.

- A second pass used stepped per-corner fill and border strips instead of rotated textures, but this still required in-game visual tuning and remained more complex than the feature justified.

- The runtime code, defaults, GUI checkbox, and profile keys were removed. Only investigation notes remain.


## Practical Constraints

- A visible texture can look rounded or chamfered, but the underlying WoW frame hitbox remains rectangular.

- Lua-drawn strips are sensitive to UI scale, pixel snapping, and frame size.

- Rotated 1px textures are not a reliable path for crisp 45-degree borders.

- If this feature is revisited, avoid hand-drawn texture-strip geometry unless there is a strong reason.


## Better Future Paths

- Use a small custom white/neutral `.tga` or `.blp` background asset with alpha-shaped corners, then tint it with `SetVertexColor`.

- Use WoW's newer texture slicing API, especially `Texture:SetTextureSliceMargins(left, top, right, bottom)`, to nine-slice one neutral texture.

- Use a traditional NineSlice set with separate corner, edge, and center textures if more control is needed.

- Consider mask textures if a single colored texture plus alpha mask fits the final design, but this still requires a mask asset.


## NineSlice Explanation

NineSlice is a resizable-panel texture technique. One panel is treated as nine regions:

```text
top-left corner     top edge      top-right corner
left edge           center        right edge
bottom-left corner  bottom edge   bottom-right corner
```

When the panel resizes, corners stay fixed, edges stretch or tile along one axis, and the center fills the remaining area. This keeps rounded or chamfered corners from distorting.

For Aura Frames, the cleanest version would likely be one neutral/tintable texture with slice margins, or a tiny neutral NineSlice texture set.


## Internet Findings

- Warcraft Wiki documents `TextureBase:SetTextureSliceMargins`, added in patch 10.2.0, as the preferred modern way to enable nine-slice texture rendering on a texture:
  https://warcraft.wiki.gg/wiki/API_TextureBase_SetTextureSliceMargins

- Masque is the closest open addon ecosystem found for shaped WoW UI texture assets, but it targets button/icon skins rather than resizable plain panels:
  https://addons.wago.io/addons/masque

- Masque skin examples such as Cirque and Apathy contain shaped texture assets and skin definitions, but their art is sized and authored for buttons:
  https://github.com/SFX-WoW/Masque_Cirque
  https://github.com/SFX-WoW/Masque_Apathy

- The Masque skin list is useful for surveying existing shaped UI approaches:
  https://github-wiki-see.page/m/SFX-WoW/Masque/wiki/Skin-List

- Gethe's WoW UI texture mirror is useful for studying Blizzard texture organization, but those files should be treated as Blizzard assets/reference material, not open-source vendorable assets:
  https://github.com/Gethe/wow-ui-textures


## Current Recommendation

Do not re-add a chamfer/rounded background option until there is a dedicated tintable asset plan.

Best revisit path:

1. Create a tiny custom neutral texture or NineSlice asset specifically for LsTweeks.

2. Verify the asset can be tinted cleanly with `SetVertexColor`.

3. Prototype it on one Aura Frame background only.

4. Test at multiple UI scales and frame sizes before adding saved variables or settings UI.
