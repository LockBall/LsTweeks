## Potential Bugs To Verify
1. Post-reset stale DB subtable bindings across the whole Audio Volumes GUI. ARM reset runs `table.wipe(db)` + `deep_copy_into(defaults, db)` (`functions/module_reset.lua:252-253`), which keeps the top-level `audio_volumes` table but replaces every subtable (`targets.*`, `fishing_focus`, `combat_volumes`, `quiet_custom`, `custom_situations`). Settings panels are built once per session and never rebuilt (`core/main_frame.lua:230-236`; the "cleared by reset" comment there has no implementing code). GUI closures capture pre-reset subtables: `target_db` in `BuildSoundTargetSliderPanel` (`av_gui.lua:130`, written by the slider callback at 209-227 and both checkboxes at 248-277), `focus_db`/`combat_db`/`quiet_custom_db` in `BuildSituationsTab` (`av_gui_situations.lua:114-116`), and situation slider value tables via `entry.db` (`av_gui_situations.lua:464`; `CreateSliderWithBox` writes `db_table[db_key]`, `functions/slider_with_box.lua:147,208,218`). After a reset with the GUI open, `on_reset_complete` resyncs displayed values from the fresh DB, but every subsequent slider/checkbox edit writes an orphaned table while `apply_audio_volumes()`/`resync_*` read the fresh DB, so post-reset edits silently neither apply nor save. The Situations left list is also not rebuilt after reset (`rebuild_situation_list` is a tab-local, never called from `av_main.lua` `on_reset_complete`), so deleted custom Quick Picks stay listed with panels bound to dead tables. In-game check: open GUI, ARM reset from General, then move a Specifics slider — the CVar/mute state and SavedVariables should not change.
2. `read_channel_percent` ignores an active manual Quick Pick. The read gate checks only `M._fishing_focus_active or M._combat_volumes_active` (`av_logic_situations.lua:79`) while the matching write path also checks `M._manual_situation_active_key` (`av_logic_situations.lua:410`). While a Quick Pick is enabled (which is indefinite by design): Normal Volumes sliders display the Quick Pick's temporary CVar values instead of the cached normal profile; dragging a Normal slider writes the cache but the next `refresh_current_values()` (`av_gui_situations.lua:255-270`, called on every tab select) snaps the display back to the temporary value; `copy_current_sound_channels_to_situation/fishing/combat` (`av_logic_situations.lua:377-400`), `create_custom_situation` seeding (`av_logic_situations.lua:217`), and first-run init-from-current in `get_combat_volumes_db`/`get_fishing_focus_db` all capture temporary Quick Pick values instead of normal ones. Module memory documents cache-write behavior for active temporary situations; the read side only honors it for the two triggered situations. Fix: include `M._manual_situation_active_key` in the line 79 gate.
3. Normal-panel test-sound preview writes CVars, deviating from the memory rule "Normal Volumes preview must not write CVars". `play_situation_preview("current", test_sound_key)` with a test sound key skips the bobber shortcut (`av_logic_situations.lua:477`), gets `profile_db = nil`, but still caches all five CVars and schedules the 2s restore (`av_logic_situations.lua:494-512`). Normally it restores identical values, but if the user moves a Normal Volumes slider inside the 2s window the delayed restore reverts the CVar underneath the GUI. Fix: skip the cache/timer when `profile_db` is nil.
4. Preview-restore race with Normal slider edits generally: `set_current_sound_channel_percent` (`av_logic_situations.lua:406-416`) neither cancels nor routes through `_fishing_bobber_preview_cached`, so any situation preview's pending 2s restore can overwrite a Normal CVar edit made during the window (only when no situation is active; the situation-active path writes the cache instead). `read_channel_percent` also reads live preview values during the 2s window. Low severity, verify-in-game.


## Latent Traps
1. `handle_event` returns after the first playable slot (`av_logic_main.lua:201-214`); every created slot has either `path` or `soundkit_id`, so slot 1 always plays and any later slots are unreachable. Fine today because `SOUND_EVENT_TARGETS` maps one target per event, but a future event shared by two targets silently plays only one. Either loop without the early returns or comment the single-slot assumption where the cache is built.
2. `set_manual_situation_enabled("fishing"/"combat", true)` (`av_logic_situations.lua:310-324`) resolves a profile db so it passes the guard, then disables every Quick Pick and enables nothing because triggered keys are absent from `get_manual_situation_entries()`. Unreachable from current callers (GUI trigger checkboxes and the minimap menu pass Quick Pick keys only); add an early reject for non-manual keys or a comment.
3. Inconsistent `M.controls` key sanitization for custom situations: enabled-checkbox keys use `key:gsub("[^%w_]", "_")` while slider keys keep the raw `custom:N` (`av_gui_situations.lua:43` vs `:46`, and `:387` vs `:473`). Both reader/writer pairs match today, but the mixed convention invites a future mismatch; controls of deleted situations also linger in `M.controls` and are only skipped because `sync_temporary_profile_controls` iterates the DB.
4. Enabling Fishing Focus while already channeling Fishing does not apply the profile until the next cast: `sync_fishing_focus_events` (`av_logic_situations.lua:710-743`) has no mid-channel equivalent of the combat path's `InCombatLockdown()` re-apply (`av_logic_situations.lua:778-780`). A `UnitChannelInfo("player")` spell-ID check could close the gap; verify-in-game whether it matters in practice.
5. Post-delete selection has two owners: `delete_custom_situation` sets `db.last_situation_key = "fishing"` (`av_logic_situations.lua:233`) and the GUI then overwrites it via `select_situation` with `quiet_custom`/fallback (`av_gui_situations.lua:578-580`). Final state is consistent only because the GUI writes last; pick one owner.


## Optimization Candidates
1. `apply_audio_volumes()` runs on every slider step during drag (`av_gui.lua:191-229` calls it at three exits): a full mute/unmute pass over all targets plus `rebuild_event_cache()` plus `sync_registered_events()` per 5% step, while the audible preview is already debounced at 0.12s (`av_logic_main.lua:183`). The mute decision only depends on `sound_off`/`use_original`, not the preset level, so mid-drag calls are redundant `MuteSoundFile` churn; debounce the apply alongside the preview or skip it when only `preset` changed between non-off values.
2. `resync_situation_runtime` on a Quick Pick slider change re-applies the active situation even when the edited situation is not the active one (`av_gui_situations.lua:285-293` routes to `resync_manual_situation_profile`, which applies `M._manual_situation_active_key` unconditionally): five redundant `SetCVar` writes per drag step while editing an inactive Quick Pick. Early-out when the edited entry key is not the active key.
3. `get_manual_situation_entries()` (`av_logic_situations.lua:279-299`) rebuilds the entry list and re-runs the per-channel clamp/validate loops in `get_situation_profile_db` on every call; `set_manual_situation_enabled` triggers it twice (directly and via `sync_manual_situation_profile`). Settings-time only, so minor.
4. `handle_delete_situation` calls `M.get_situation_profile_db(delete_key)` twice for the `was_enabled` check (`av_gui_situations.lua:570-573`); fold into one local.


## Minor Cleanups
1. `M._fishing_focus_cached` is a write-only legacy alias (`av_logic_situations.lua:412,527,544`) — never read anywhere in the repo; remove it or comment why it is kept.
2. Inner `situation_grid` local in `create_situation_sliders` (`av_gui_situations.lua:452`) shadows the tab-level `situation_grid` (`av_gui_situations.lua:148`); rename one.
3. Normal-slider defaults seeding has three owners: the build loop (`av_gui_situations.lua:295-299`), `refresh_current_values` (`av_gui_situations.lua:255-270`), and the slider on_change (`av_gui_situations.lua:313-314`) all write `focus_defaults`/`combat_defaults`/`quiet_custom_defaults`. One helper would satisfy the constants-owned-in-one-place rule.
4. `col_align = { "left", "left", "left", "left", "left" }` hardcodes five entries while `column_count = slider_count` is derived from `FISHING_FOCUS_CHANNELS` (`av_gui_situations.lua:159,281`); build the align table from `slider_count`.
5. `play_original_file` unmute loop (`av_logic_main.lua:72-74`) is redundant on its only reachable path (use_original targets are already unmuted by `apply_audio_volumes`); keep as defense with a comment or drop it.
6. `local profile_db = nil` immediately reassigned (`av_logic_situations.lua:451-452`); collapse to one line.
7. `set_quick_pick_from_menu` syncs controls twice, immediately and via `C_Timer.After(0)` (`av_logic_situations.lua:364-373`); if the deferred pass covers a menu-close timing issue, comment it, otherwise drop one.


## Reviewed And Confirmed Deliberate
Checked against `proj_mem/modules/audio_volumes.md`; do not re-flag without new evidence.
- Combat end restores normal volumes rather than returning to Fishing Volumes even if still channeling: `apply_combat_volumes` clearing `_fishing_focus_active` (`av_logic_situations.lua:656`) matches the documented combat-exits-fishing priority. The reverse case (fishing channel starting mid-combat sets the flag and resumes after combat if still channeling) also resolves correctly through `apply_active_sound_channel_profile` priority order.
- Event hot path matches the Event Cache And Performance section: `handle_event` reads only pre-resolved `M._event_cache` slots with no DB/preset/path work (`av_logic_main.lua:193-215`), Off and Original targets create no slots (`av_functions.lua:139-156`), `sync_registered_events` diffs registrations (`av_logic_main.lua:217-244`), and mute/unmute stays outside the hot path. No polling or OnUpdate anywhere in the module.
- `PlaySound(soundKitID, "SFX")` string-channel calls with inline Ketho `param-type-mismatch` suppressions: verified in-game per memory; do not convert to numeric `UISoundSubType`.
- Bloodlust test sound played via `PlaySoundFile(568812)` file-ID branch (`av_logic_situations.lua:505-506`): verified in-game per memory; `PlaySound(568812)` does not work.
- `achievmentsound1` spelling matches Blizzard's original asset naming and the on-disk replacement folder; do not correct in isolation.
- Fishing events registered player-only via `RegisterUnitEvent` and only while enabled; combat events only while enabled; disabled sync branches restore and unregister without creating frames or initializing situation DB values — matches the memory's disabled-sync and lazy-init rules.
- `apply_active_sound_channel_profile` calls `restore_bobber_preview_profile()` before writing runtime CVars (`av_logic_situations.lua:548`) — the documented preview-vs-active-situation ordering rule.
- `create_custom_situation` seeding `last_situation_key` so new Quick Picks open immediately in the unified Situations tab — documented GUI behavior.
- Sound API upvalues resolved once at file load in `av_logic_main.lua:8-12` and reused at call sites — documented ownership rule.
