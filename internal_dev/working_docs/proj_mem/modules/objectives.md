## Settings And Defaults
Important `objectives` keys:
- `collapse_all`: when true, starts Blizzard's All Objectives tracker collapsed.
- `collapse_campaign`: when true, starts Blizzard's Campaign module in the All Objectives tracker collapsed.
- `collapse_quests`: when true, starts Blizzard's Quests module in the All Objectives tracker collapsed.
- `collapse_achievements`: when true, starts Blizzard's Achievements module in the All Objectives tracker collapsed.
- `show_quest_log_count`: when true, shows accepted quest log count/capacity in the Quests objective section title.
- `show_quest_log_count_on_hover`: when true, shows the Quests section count only while hovering the Quests title.
- `show_tracked_achievement_count`: when true, shows tracked achievement count/capacity in the Achievements objective section title.
- `show_tracked_achievement_count_on_hover`: when true, shows the Achievements section count only while hovering the Achievements title.
- `customize_background`: when true, sets Blizzard's Objective Tracker Edit Mode opacity to `background_alpha`; when false, sets that opacity to 0.
- `background_color_enabled`: when true, shows the addon-owned center color block and enables the color picker. Missing legacy values are migrated to true only when `customize_background` is true and `background_color` is already non-default.
- `background_alpha`: alpha applied to Blizzard's own All Objectives `NineSlice` background. Default `0.5`.
- `background_color`: RGB/alpha for the owned center color block over the All Objectives background. Default/reset is 25% gray at 75% alpha: `{ r = 0.25, g = 0.25, b = 0.25, a = 0.75 }`.
- `objective_tracker_offset_x` / `objective_tracker_offset_y`: saved All Objectives tracker offsets from its captured startup center anchor.
- `objective_tracker_move_mode`: when true, allows dragging All Objectives and highlights the All Objectives header text.
- `objective_tracker_snap_to_grid`: when true, rounds All Objectives offsets to the module's grid size.
- `objective_tracker_border`: optional explicit border override independent from `background_color_enabled`. When unset, the border is inferred on for non-default `background_color` and off for default `background_color`.


## Runtime Notes
- `modules/objectives/ob_defaults.lua` owns default DB values plus shared `M.get_db()` / `M.is_runtime_enabled()` helpers. `modules/objectives/ob_position.lua` owns All Objectives tracker offsets, move mode, snap-to-grid, Position settings UI, and position status fields. `modules/objectives/ob_auto_collapse.lua` owns Auto-Collapse tracker metadata, settings UI, collapse runtime, and status fields. `modules/objectives/ob_section_count.lua` owns Section Count settings, count APIs, title sync, event registration, and status fields. `modules/objectives/ob_background.lua` owns background sizing, background color/opacity, owned border, Objective Tracker background hooks, and background status fields. `modules/objectives/ob_main.lua` owns module lifecycle, settings-page composition, event bootstrap, and module status aggregation.
- Objective settings group chrome uses addon-wide `addon.CreateSettingsGroup()` from `functions/ui_helpers.lua`, matching the Audio Volumes gold outline and 24px grey title bar. Use it for the group/title shell instead of hand-building repeated BackdropTemplate/title blocks in feature files.
- Objectives settings page order is Position, Background, Auto-Collapse, then Section Count below Auto-Collapse; all groups use the shared settings grid helpers for control placement. Position, Background, Auto-Collapse, and Section Count keep content-sized group widths.
- Objectives intentionally has no module reset panel; the page has narrow behavior groups, and the Settings module enabler already covers module-level disabling.
- Auto-Collapse tracker metadata lives in `TRACKER_DEFS` in `modules/objectives/ob_auto_collapse.lua`; it owns DB keys, labels, status keys, global frame names, and top-to-bottom UI order. Current order: All Objectives, Campaign, Quests, Achievements. The settings UI shows these inside an outlined `Auto-Collapse` group, with Campaign, Quests, and Achievements at one shared child indent under All Objectives.
- Blizzard's `ObjectiveTrackerFrame`, `CampaignQuestObjectiveTracker`, `QuestObjectiveTracker`, and `AchievementObjectiveTracker` all expose `SetCollapsed`/`IsCollapsed`. Auto-Collapse calls `SetCollapsed(true)` when LsTweeks applies settings, then leaves Blizzard's own manual expand/collapse behavior alone.
- Unchecking an Auto-Collapse option calls that tracker frame's `SetCollapsed(false)` once so the section reopens immediately. Disabling the Objectives module does not force-expand sections; later tracker state is left to Blizzard/user actions.
- `ObjectiveTrackerFrame.NineSlice` is Blizzard's All Objectives background. The Objectives module post-hooks tracker updates only to correct the collapsed All Objectives container case and to let Blizzard priority content surface itself: when `ObjectiveTrackerFrame` itself is collapsed, LsTweeks forces the background to the header. It also watches Blizzard changes to `ObjectiveTrackerFrame.NineSlice:SetPoint()`; if Blizzard anchors the background under a `hasDisplayPriority` module while All Objectives is collapsed, LsTweeks treats that as priority content claiming space and force-expands the tracker after the collapse grace window. Ordinary section anchors are corrected back to the header and reported as `bg_blocked_anchor` instead of reopening All Objectives. When the tracker is expanded, Blizzard's normal Objective Tracker layout is fully in control. Disabling the module calls Blizzard's tracker update once to hand background layout back.
- The Position settings group owns move-mode/snap controls and X/Y offset sliders because those controls affect the whole All Objectives tracker frame, not either background layer. The Background settings group owns separate `WoW BG` and `Custom BG` toggles, Blizzard background alpha, `background_color`, and a Border checkbox. `WoW BG` mirrors Blizzard's Objective Tracker Edit Mode opacity control: checked sets opacity to `background_alpha`; unchecked sets opacity to 0. `background_alpha` controls Blizzard's real Objective Tracker opacity via Edit Mode when available, falls back through `ObjectiveTrackerManager:SetOpacity()`, and hooks the inner slider `OnValueChanged` for immediate drag preview because the shared slider callback is debounced. `background_color` controls an owned rectangular center color block because the current client did not expose a stable rounded-corner RGB tint path for the `NineSlice`; its alpha is edited in the shared color picker popup. `Custom BG` gates only the owned color block and color picker; Border stays enabled/clickable and can show independently. The owned color block lives on the tracker parent for alpha independence, sits behind `NineSlice` so Blizzard's line art stays visible, and hides `NineSlice.Center` while enabled so the color shows through. The color picker reset restores 25% gray at 75% alpha and enables/checks Border, including the border offset adjustment. `/lst status objectives` reports `bg_*` background diagnostics plus `bg_region_<n>_*` diagnostics for region name, type, draw layer, size, alpha, vertex color, blend, desaturation, texture, and atlas. Disabling the Objectives module hides owned color/border overlays, writes full Blizzard background opacity through Edit Mode when available, and hands layout back to Blizzard.
- Objective Tracker Edit Mode opacity uses a `0..100` display/storage value, while LsTweeks `background_alpha` is `0..1`; convert at the API boundary only. `bg_edit_mode_state=edit_mode:<n>` reports the Edit Mode/manager percent written, and `bg_nineslice_alpha=<n>` reports the live `NineSlice` alpha. If `WoW BG` is checked and status shows `bg_wow_enabled=true`, `bg_alpha=0.5`, but `bg_edit_mode_state=edit_mode:0` / `bg_nineslice_alpha=0`, the checked path is still passing an explicit zero override. The 2026-06-28 regression was caused by `show_blizzard_background and nil or 0`; never use Lua `and/or` ternary form when the intended true branch is `nil`.
- Background sizing treats `ObjectiveTrackerFrame` collapsed state as header-only using `IsCollapsed()`, `isCollapsed`, or `collapsed`, clears the stale bottom anchor when possible before retargeting to the header, and schedules one short follow-up sync after tracker collapse changes to avoid races with Blizzard's own late tracker update. The background `SetPoint` hook corrects ordinary collapsed anchors immediately, with a guard around LsTweeks-owned `SetPoint` calls; the collapse grace window delays priority force-expansion only, not ordinary-anchor cleanup. `/lst status objectives` reports `bg_anchor`; `bg_anchor=blizzard` means LsTweeks intentionally did not override the expanded layout. It also reports `bg_point_hook`, `bg_force_expand`, `bg_blocked_anchor`, `bg_force_expand_grace`, and `bg_module_<n>_*` diagnostics for Blizzard module name, priority, shown/collapsed/displayable state, public/raw content heights, raw `hasContents` / tried / skipped fields, state, and last block when investigating world-event objectives.
- Protected Objective Tracker mutations are combat-deferred through `M.defer_objectives_combat_update()` in `ob_main.lua`. Position apply/restore, move-mode tracker changes, Auto-Collapse tracker collapse/expand, background color/opacity writes, background anchor correction, border sync, and tracker `Update()` replay after `PLAYER_REGEN_ENABLED`; settings values still save immediately during combat.
- Section Count checkboxes enable counters on Blizzard's own module header text, with a per-section `On Hover` checkbox to make each counter hover-only. Quests show bare `x / y` accepted quest log capacity by scanning `C_QuestLog.GetInfo()` over `C_QuestLog.GetNumQuestLogEntries()` and excluding headers, hidden/internal entries, tasks, bounties, and world quests before comparing against `C_QuestLog.GetMaxNumQuestsCanAccept()`. Do not use the second `GetNumQuestLogEntries()` return directly for this display; it can include entries that exceed the accepted-quest cap. Achievements show bare `x / y` tracked achievement capacity from `C_ContentTracking.GetTrackedIDs(Enum.ContentTrackingType.Achievement)` plus `Constants.ContentTrackingConsts.MaxTrackedAchievements`.
- Section Count display formatting is centralized in `ob_section_count.lua` constants `COUNT_TEXT_FORMAT` and `TITLE_COUNT_FORMAT`.
- When a Section Count checkbox is off, or when `On Hover` is enabled and the header is not hovered, Objectives must not keep rewriting that section title. It should only restore Blizzard's base title once if LsTweeks had previously applied a counter, so it does not fight Blizzard refreshes or another title addon while the option is disabled or idle.
- Section Count is event-driven and does not hook objective tracker layout/update methods. It registers quest title events only while `show_quest_log_count` is true and achievement title events only while `show_tracked_achievement_count` is true. Count scans happen on owned events for always-visible counters, or only while the matching section header is hovered for hover-only counters; syncs cache the displayed count signature and skip `SetText` when the count text has not changed.
- Current generated Blizzard docs list the tracked achievement cap as 10. Quest-log capacity is read through `C_QuestLog.GetMaxNumQuestsCanAccept()` instead of hardcoded.
- Initialization can race `ObjectiveTrackerManager:Init()`, which runs after both `PLAYER_ENTERING_WORLD` and `VARIABLES_LOADED`. Remove the `ADDON_LOADED` or delayed `PLAYER_ENTERING_WORLD` apply paths only after in-game testing proves they are redundant.
