# Objectives Memory
Important `objectives` keys:
- `collapse_all`: when true, starts Blizzard's All Objectives tracker collapsed.
- `collapse_campaign`: when true, starts Blizzard's Campaign module in the All Objectives tracker collapsed.
- `collapse_quests`: when true, starts Blizzard's Quests module in the All Objectives tracker collapsed.
- `collapse_achievements`: when true, starts Blizzard's Achievements module in the All Objectives tracker collapsed.
- `show_quest_log_count`: when true, shows accepted quest log count/capacity in the Quests objective section title.
- `show_quest_log_count_on_hover`: when true, shows the Quests section count only while hovering the Quests title.
- `show_tracked_achievement_count`: when true, shows tracked achievement count/capacity in the Achievements objective section title.
- `show_tracked_achievement_count_on_hover`: when true, shows the Achievements section count only while hovering the Achievements title.
- `background_color`: tint/alpha applied to Blizzard's All Objectives `NineSlice` background while the Objectives module is enabled. Default `{ r = 1, g = 1, b = 1, a = 1 }` leaves Blizzard's texture color unchanged.


## Runtime Notes
- `modules/objectives/ob_defaults.lua` owns default DB values plus shared `M.get_db()` / `M.is_runtime_enabled()` helpers. `modules/objectives/ob_auto_collapse.lua` owns Auto-Collapse tracker metadata, settings UI, collapse runtime, and status fields. `modules/objectives/ob_section_count.lua` owns Section Count settings, count APIs, title sync, event registration, and status fields. `modules/objectives/ob_main.lua` owns background sizing, module lifecycle, settings-page composition, and module status aggregation.
- Objectives intentionally has no module reset panel; the page has narrow behavior groups, and the Settings module enabler already covers module-level disabling.
- Auto-Collapse tracker metadata lives in `TRACKER_DEFS` in `modules/objectives/ob_auto_collapse.lua`; it owns DB keys, labels, status keys, global frame names, and top-to-bottom UI order. Current order: All Objectives, Campaign, Quests, Achievements. The settings UI shows these inside an outlined `Auto-Collapse` group, with Campaign, Quests, and Achievements indented under All Objectives.
- Blizzard's `ObjectiveTrackerFrame`, `CampaignQuestObjectiveTracker`, `QuestObjectiveTracker`, and `AchievementObjectiveTracker` all expose `SetCollapsed`/`IsCollapsed`. Auto-Collapse calls `SetCollapsed(true)` when LsTweeks applies settings, then leaves Blizzard's own manual expand/collapse behavior alone.
- Unchecking an Auto-Collapse option calls that tracker frame's `SetCollapsed(false)` once so the section reopens immediately. Disabling the Objectives module does not force-expand sections; later tracker state is left to Blizzard/user actions.
- `ObjectiveTrackerFrame.NineSlice` is Blizzard's All Objectives background. The Objectives module post-hooks tracker updates only to correct the collapsed All Objectives container case and to let Blizzard priority content surface itself: when `ObjectiveTrackerFrame` itself is collapsed, LsTweeks forces the background to the header. It also watches Blizzard changes to `ObjectiveTrackerFrame.NineSlice:SetPoint()`; if Blizzard anchors the background under a `hasDisplayPriority` module while All Objectives is collapsed, LsTweeks treats that as priority content claiming space and force-expands the tracker after the collapse grace window. Ordinary section anchors are corrected back to the header and reported as `background_blocked_anchor` instead of reopening All Objectives. When the tracker is expanded, Blizzard's normal Objective Tracker layout is fully in control. Disabling the module calls Blizzard's tracker update once to hand background layout back.
- The Background settings group owns a color picker for `background_color`. It tints each texture region returned by `ObjectiveTrackerFrame.NineSlice:GetRegions()` and caches the color signature so routine tracker updates do not rewrite unchanged colors. Disabling the Objectives module restores `{ r = 1, g = 1, b = 1, a = 1 }` before handing the layout back to Blizzard.
- Background sizing treats `ObjectiveTrackerFrame` collapsed state as header-only using `IsCollapsed()`, `isCollapsed`, or `collapsed`, clears the stale bottom anchor when possible before retargeting to the header, and schedules one short follow-up sync after tracker collapse changes to avoid races with Blizzard's own late tracker update. The background `SetPoint` hook corrects ordinary collapsed anchors immediately, with a guard around LsTweeks-owned `SetPoint` calls; the collapse grace window delays priority force-expansion only, not ordinary-anchor cleanup. `/lst status` reports `background_anchor`; `background_anchor=blizzard` means LsTweeks intentionally did not override the expanded layout. It also reports `background_point_hook`, `background_force_expand`, `background_blocked_anchor`, `background_force_expand_grace`, and `background_module_<n>_*` diagnostics for Blizzard module name, priority, shown/collapsed/displayable state, public/raw content heights, raw `hasContents` / tried / skipped fields, state, and last block when investigating world-event objectives.
- Section Count checkboxes enable counters on Blizzard's own module header text, with a per-section `On Hover` checkbox to make each counter hover-only. Quests show bare `x / y` accepted quest log capacity by scanning `C_QuestLog.GetInfo()` over `C_QuestLog.GetNumQuestLogEntries()` and excluding headers, hidden/internal entries, tasks, bounties, and world quests before comparing against `C_QuestLog.GetMaxNumQuestsCanAccept()`. Do not use the second `GetNumQuestLogEntries()` return directly for this display; it can include entries that exceed the accepted-quest cap. Achievements show bare `x / y` tracked achievement capacity from `C_ContentTracking.GetTrackedIDs(Enum.ContentTrackingType.Achievement)` plus `Constants.ContentTrackingConsts.MaxTrackedAchievements`.
- Section Count display formatting is centralized in `ob_section_count.lua` constants `COUNT_TEXT_FORMAT` and `TITLE_COUNT_FORMAT`.
- When a Section Count checkbox is off, or when `On Hover` is enabled and the header is not hovered, Objectives must not keep rewriting that section title. It should only restore Blizzard's base title once if LsTweeks had previously applied a counter, so it does not fight Blizzard refreshes or another title addon while the option is disabled or idle.
- Section Count is event-driven and does not hook objective tracker layout/update methods. It registers quest title events only while `show_quest_log_count` is true and achievement title events only while `show_tracked_achievement_count` is true. Count scans happen on owned events for always-visible counters, or only while the matching section header is hovered for hover-only counters; syncs cache the displayed count signature and skip `SetText` when the count text has not changed.
- Current generated Blizzard docs list the tracked achievement cap as 10. Quest-log capacity is read through `C_QuestLog.GetMaxNumQuestsCanAccept()` instead of hardcoded.
- Initialization can race `ObjectiveTrackerManager:Init()`, which runs after both `PLAYER_ENTERING_WORLD` and `VARIABLES_LOADED`. Remove the `ADDON_LOADED` or delayed `PLAYER_ENTERING_WORLD` apply paths only after in-game testing proves they are redundant.
- In-game verification passed after the Auto-Collapse group work: checked All Objectives, Campaign, Quests, and Achievements sections start collapsed after reload/login with tracked entries, remain manually expandable/collapsible through Blizzard tracker controls, and reopen once when their option is unchecked. No blocked-action/taint dialogs were observed during the verification pass.
- Current Section Count in-game smoke testing appeared good before the display-mode adjustment: the feature no longer flickered when its checkbox was off, and title counts updated through owned events rather than movement/tracker churn. Re-test always-visible and hover-only display behavior with `modules/objectives/obj_scratch.md` before final package validation and any near-cap quest-count edge checks are complete.
